Projet Club Photo

Première année

LIRE

Objectif : Développer une interface web pour le Club Photo de Limoges, permettant la gestion des articles, des photos, et des informations des photographes

Compétences mobilisées :

  • B1.1 Gérer le patrimoine informatique : mise en place d'une BDD, code structuré
  • B1.2 Répondre aux incidents : correction d’erreurs SQL
  • B1.3 Développer la présence en ligne : site attrayant et fonctionnel
  • B1.4 Travailler en mode projet : gestion par étapes, documentation
  • B1.6 Développement professionnel : montée en compétences PHP, SQL, sécurité etc

Résultat : J’ai livré une application web qui permet aux membres du club de gérer facilement les photos et articles, et d’administrer les informations des photographes. J’ai appris à concevoir une application multi-parties (gestion d’éléments liés entre eux), à sécuriser les interactions utilisateurs/base de données, et à proposer une navigation fluide. Parmi les principales difficultés : assurer la cohérence des liens entre les modules (photos, articles, photographes) et éviter les failles SQL.

06

Projet Bibliothèque

Deuxième année

LIRE

Objectif : Développer une application mobile multiplateforme (Flutter/Dart) pour gérer le catalogue d’une bibliothèque. L’application permet aux bibliothécaires d’ajouter, modifier, supprimer et consulter les livres et auteurs, avec une synchronisation en temps réel via Firebase.

Compétences mobilisées :

  • B1.1 Gérer le patrimoine informatique : gestion de la base de données Firebase, structuration des entités (livres, auteurs) et mise en place des règles de sécurité.
  • B1.3 Développer la présence en ligne : création d’une interface intuitive et responsive pour renforcer l’image numérique de la bibliothèque.
  • B1.4 Travailler en mode projet : adoption de l’architecture MVVM, planification des développements et usage de Git pour le versioning.
  • B1.6 Organiser son développement professionnel : montée en compétences sur Flutter, Firebase et les tests d’application mobile.

Résultat : J’ai réalisé une application fonctionnelle testée sur simulateur et appareils physiques, capable de gérer les ressources en temps réel. J’ai appris à structurer un projet mobile complexe, à configurer une authentification sécurisée et à anticiper les besoins de déploiement multi-plateforme. Les principales difficultés ont été liées à la synchronisation en temps réel et à la configuration des règles Firebase.
